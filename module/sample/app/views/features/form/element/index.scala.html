@( library: String, sample: String, documentation: String ) @import views.features.form.element.{input, textarea}

Input

The most general way to create an <input /> is to call com.taig.bootstrapper.Template.Form.Element.Input. But go ahead ad explore the sub-objects. They come in pretty handy.

@input.Signature
Text
@input.text.Signature @input.text.Usage
Password

Creating a password field works pretty much just like creating a text field, except you have to call @@Input.Password instead of @@Input.Text.

Hidden

Surprisingly the same thing applies for hidden fields. Give @@Input.Hidden a go. But of course you don't have the placeholder parameter here.

@input.hidden.Signature
Checkbox

A checkbox, however, has a slightly different method signature than the previous input fields.

@input.checkbox.Signature

But as you can see, the main difference is that the paramter placeholder got replaced by checked. Pretty straight forward, isn't it?

@input.checkbox.Usage
Radio

Well, it works just as @@Input.Checkbox. Just make sure to call @@Input.Radio instead. The apply method's signatures are equal.
