@import views.features.button._

Basics

To create a button you have to provide a label (may be None) and a String URL as a target. Alternatively you may pass a Call from your router as an URL (and you are highly encouraged to do this whenever possible) but then you need to make sure that the implicit conversion com.taig.bootstrapper.Template.callToString is in your view's scope. All further parameters are optional.

@Signature

The Style, Size and Icon objects are named like the respective classes in Twitter Bootstrap. Parameters that expect an Option type (in this case label and icon) are backed up by implicit conversions to simplify a button's instantiation.

@Usage

If you pass a String like "tag" or "heart" as icon parameter the icon's color (black or white) will be chosen automatically according to the button's style.

@Colors

Common Use Cases

Some frequently used buttons are already predefined. You'll find them here: com.taig.bootstrapper.Template.Button.{Add, ..., Save}.

@CommonUseCases
The labels on these buttons support Play's internationalization tools. Have a look at the configuration files to add your project's language.

Not So Common Use Cases

Note the parameter compact: If you set compact to true the label text will be moved to the button's title (hover it with your mouse to have a look) and the button will then only display the icon.
