Play Bootstrapper

A module for the Play Framework 2 (Scala) that helps you at keeping your Twitter Bootstrap-views clean and free from boilerplate code.

Usage

Using the Play Bootstrapper module should in general be limited to the Template file. If you find yourself importing other files into your views you're either extending the module's functionality or you are most likely on the wrong track. Therefore importing com.taig.bootstrapper.Template._ on the top of your view is a good way to get started. 

@*************************
* Your view's signature. *
*************************@
@( args: Any* )

@*********************
* Import statements. *
*********************@
@import com.taig.bootstrapper.Template._

<p>Your HTML code.</p>

For further code examples and insights you may take a look into this documentation page's source code that comes bundled with the module's source as sample application.

Features

All templating features aim to provide maximum flexibility to allow for a widget to enter all possible states that are supported by Twitter Bootstrap. If you find yourself in a situation where the module's behavior doesn't measure up your expectations it should be fairly simple to extend its functionality after having a look at the source. (Please make sure to send me a pull request if you improve something). Furthermore there are plenty of common use case widgets (e.g. a cancel button) to keep method signatures short and simple whenever possible.

Buttons

The labels on the common use case buttons support Play's internationalization tools. Have a look at the configuration files.
